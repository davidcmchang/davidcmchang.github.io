# TowardingWeb - 專案規則與開工指引

本專案是一個全新初始化的 Web 專案。

## 🛠️ 開工與收工流程

### 開工流程
1. 讀取 `ANTIGRAVITY.md` 瞭解最新專案規則與狀態。
2. 讀取專案筆記（如 `README.md` 或其他筆記文件）掌握開發進度。
3. 執行 `git status` 與檢查最近的 commit 紀錄。
4. 向用戶回報當前狀態，並建議下一步的行動方案。
5. **不自動**進行 `git pull`、`commit` 或 `push`，必須徵得用戶同意或由用戶執行。

### 收工流程
1. **檢查敏感資料**：確保沒有將 API Keys、Tokens、密碼或個人隱私資訊寫入程式碼中。
2. **更新專案筆記**：在 `README.md` 或專案筆記中更新「已完成事項」、「下一步計畫」以及「遇到的問題/踩坑紀錄」。
3. **更新規則**：僅在專案規則或架構有變動時，才更新本 `ANTIGRAVITY.md`。
4. **檢查 Git 狀態**：執行 `git status` 與 `git diff`，確認修改範圍。
5. **精確 Commit**：只將本次開發相關的檔案加入暫存區（避免直接使用 `git add .` 一次全加），確認無誤後再進行 commit 與 push。
